# Curator

Use this skill when a repeated workflow deserves a new repo-local skill, or when
existing skills have become stale, duplicated, or misleading.

## Procedure

1. Search `SKILLS.md`, `skills/`, `MEMORY.md`, and `AGENTS.md` for overlapping guidance.
2. Prefer updating an existing skill over adding a new one.
3. Keep `SKILLS.md` as a compact catalog; put the detailed procedure in `skills/<name>/SKILL.md`.
4. Keep skills small, command-oriented, and specific to Converge.
5. Remove or revise stale guidance when tool behavior, repo conventions, or user preferences change.
6. Record meaningful operating-practice changes in `AGENTS.md` and, when useful, a dated `memory/logs/` entry.

## Quality Bar

A skill should save future agents real time. Do not create one for a one-off
task, a vague preference, or information that belongs in feature docs.
