# Collaborator Notes

Use this skill when a user preference, workflow expectation, or communication
cue should be retained for future Converge work.

## Procedure

1. Decide whether the note belongs in `AGENTS.md`, `MEMORY.md`, or `memory/people/`.
2. Put project-critical instructions in `AGENTS.md`.
3. Put searchable but lower-priority context in `memory/notes/` or `memory/logs/`.
4. Put collaborator preferences in `memory/people/` only when they are practical, respectful, and relevant to future repo work.
5. Keep notes concrete: include the command, path, behavior, or decision that future agents should reuse.

## Restraint

Do not record private, speculative, or irrelevant personal details. If a note
would not help future work in this repository, leave it out.
